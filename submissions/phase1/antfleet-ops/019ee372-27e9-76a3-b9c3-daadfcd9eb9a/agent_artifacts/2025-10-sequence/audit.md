# Audit: 2025-10-sequence

## Unauthenticated `delegatecall` in `Simulator` and `Estimator` allows self-destruction
- Location: `src/Simulator.sol` : `simulate` and `src/Estimator.sol` : `estimate` (via `_estimate`)
- Mechanism: Both the `Simulator` and `Estimator` contracts process arrays of user-provided `Payload.Call` structs and expose the ability to perform a `delegatecall` to an arbitrary, user-controlled address (`call.to`). The `simulate` function has no authentication checks at all. The `estimate` function appears to authenticate, but intentionally neutralizes the check by overriding `_isValidImage` to always return `true`, allowing any arbitrary signature to pass. Because the target address and data flow are fully controlled by the caller, an attacker can deploy a malicious contract whose fallback function executes `selfdestruct(payable(msg.sender))` and specify its address as `call.to`. When the `Simulator` or `Estimator` executes the `delegatecall` to this malicious contract, it executes the `selfdestruct` opcode within the context of the `Simulator` or `Estimator` contract itself.
- Impact: An attacker can remotely trigger self-destruction of the deployed `Simulator` and `Estimator` utility contracts. This causes a permanent Denial of Service (DoS) for the protocol's simulation and gas estimation features across the network. If the wallet implementations or off-chain SDKs rely heavily on these singleton endpoint contracts for transaction construction, it completely paralyzes those workflows.
